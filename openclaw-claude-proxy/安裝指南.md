# OpenClaw + Claude Code Proxy 安裝指南

用 **Claude Max 訂閱（約 $200/月）** 透過 Claude Code CLI 驅動 Telegram AI 助手（Opus 4.6），無需昂貴的 API 直連。

---

## 前置條件

1. **Claude Max 訂閱**（約 $200/月）
2. **一台 Ubuntu 22.04+ 機器**（建議 2GB+ RAM，可用 AWS EC2、VPS 或本機）
3. **已 SSH 登入該機器**（或直接在終端操作）

---

## 方式一：一鍵安裝（推薦）

在 **Ubuntu** 上執行以下指令，約 5 分鐘完成：

```bash
# 下載並執行官方一鍵腳本（需先準備 Bot Token 與 User ID）
curl -fsSL https://raw.githubusercontent.com/51AutoPilot/openclaw-claude-proxy/master/setup.sh -o setup.sh
bash setup.sh
```

執行時會詢問：
- **Telegram Bot Token**：到 Telegram 搜尋 `@BotFather` → `/newbot` → 取得 Token
- **你的 Telegram User ID**：搜尋 `@userinfobot` → 發送訊息 → 取得數字 ID

**重要：** 腳本進行到「Step 3: Claude CLI 認證」時，若尚未認證，請在**另一個終端**執行：

```bash
claude
```

瀏覽器會開啟登入頁 → 使用你的 Claude Max 帳號登入 → 完成後回到原終端按 Enter 繼續。

## 方式二：手動安裝

### Step 1：安裝 Node.js 22

```bash
curl -fsSL https://deb.nodesource.com/setup_22.x | sudo -E bash -
sudo apt-get install -y nodejs
node -v   # 應為 v22.x
```

### Step 2：安裝 PM2 與 Claude Code CLI

```bash
sudo npm install -g pm2 @anthropic-ai/claude-code
```

### Step 3：認證 Claude CLI（只需做一次）

```bash
claude
# 瀏覽器開啟 URL → 登入 Max 帳號 → 完成後 Ctrl+C
# 驗證：claude --print "hi" 應有正常回覆
```

### Step 4：部署 Proxy

**4.1** 在本機專案中已有 `openclaw-claude-proxy` 資料夾，可整份上傳到伺服器，或直接在伺服器上建立：

```bash
mkdir -p ~/openclaw-claude-proxy
# 將 server.js、package.json、ecosystem.config.js 複製到該目錄
cd ~/openclaw-claude-proxy
```

**4.2** 安裝依賴並建立 `.env`：

```bash
npm install
cp .env.example .env
# 編輯 .env，至少設定：
# API_KEY=sk-proxy-$(openssl rand -hex 16)   # 或自訂一組密鑰
nano .env
```

**4.3** 啟動 Proxy（PM2 + 開機自啟）：

```bash
pm2 start ecosystem.config.js
pm2 save
pm2 startup   # 依提示執行輸出的 sudo 指令
```

**4.4** 檢查：

```bash
curl http://localhost:3456/health
# 應回傳 {"status":"ok", ...}
```

### Step 5：安裝 OpenClaw

```bash
sudo npm install -g openclaw@latest
```

### Step 6：設定 OpenClaw

**Telegram：**

```bash
openclaw config set channels.telegram.botToken "你的BOT_TOKEN"
openclaw config set channels.telegram.dmPolicy allowlist
openclaw config set channels.telegram.allowFrom --json '["telegram:你的USER_ID"]'
```

**Gateway：**

```bash
openclaw config set gateway.mode local
```

**自訂 Provider（指向本地 Proxy）：**

```bash
# 將 PROXY_API_KEY 換成 .env 裡的 API_KEY
openclaw config set 'models.providers.claude-proxy' --json '{
  "baseUrl": "http://localhost:3456/v1",
  "apiKey": "你的PROXY_API_KEY",
  "api": "openai-completions",
  "models": [
    {"id": "claude-opus-4-6", "name": "Claude Opus 4.6"}
  ]
}'

openclaw config set agents.defaults.model.primary "claude-proxy/claude-opus-4-6"
```

### Step 7：以 systemd 啟動 OpenClaw Gateway

```bash
# 先查 openclaw 路徑（例如 /usr/local/bin/openclaw）
which openclaw

sudo tee /etc/systemd/system/openclaw.service << EOF
[Unit]
Description=OpenClaw Gateway
After=network.target

[Service]
Type=simple
User=ubuntu
WorkingDirectory=/home/ubuntu
Environment=PATH=/usr/bin:/usr/local/bin
ExecStart=$(which openclaw) gateway
Restart=always
RestartSec=5

[Install]
WantedBy=multi-user.target
EOF
# 若上面 ExecStart 沒有被替換成實際路徑，請手動改為：ExecStart=/usr/local/bin/openclaw gateway
# 並將 User、WorkingDirectory 改成你的實際使用者與家目錄

sudo systemctl daemon-reload
sudo systemctl enable openclaw
sudo systemctl start openclaw
```

## 常用指令

| 用途 | 指令 |
|------|------|
| 看 Proxy 日誌 | `pm2 logs openclaw-claude-proxy` |
| 看 Gateway 日誌 | `sudo journalctl -u openclaw -f` |
| 重啟 Proxy | `pm2 restart openclaw-claude-proxy` |
| 重啟 Gateway | `sudo systemctl restart openclaw` |

## 常見問題

| 狀況 | 可能原因 | 解法 |
|------|----------|------|
| Node 版本錯誤 | OpenClaw 需要 Node 22+ | 改用 Node 22：`setup_22.x` |
| Gateway 沒啟動 | 未設定 gateway.mode | `openclaw config set gateway.mode local` |
| Telegram 不回應 | 存取未設定 | 設 `dmPolicy: allowlist` 並加入自己的 user ID |
| 出現 Unknown model | 未用自訂 provider | 必須用 `models.providers` 註冊 claude-proxy |
| OPENAI_BASE_URL 無效 | OpenClaw 不讀此變數 | 一律用 config 設定 provider |
| Streaming 卡住 | CLI 不支援真串流 | 代理使用 simulated stream，屬正常 |
| Sandbox 擋寫入 | CLI 工作目錄權限 | 工作目錄為 $HOME，必要時調整 tools 設定 |

## 架構說明

```
Telegram → @你的Bot → OpenClaw Gateway → 本 Proxy (localhost:3456)
         → claude --print (官方 CLI) → Anthropic（Max 訂閱）
```

- 所有請求經由官方 Claude Code CLI 送出，等同在終端機前操作，不偷 Token、不違規。
- 建議在 **EC2 / VPS** 上執行，不要用存有個人資料的本機，以降低風險。
